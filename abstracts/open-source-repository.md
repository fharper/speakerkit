# Open Source: a public repository is the first step, not the finish line

Does pushing code to a public repository on GitHub or GitLab with a proper license means you are doing open source? In theory, if you follow the ten points from the open source definition of the Open Source Initiative (OSI), you are, but in reality it’s much more than that!

Being open source is not just about the code. It’s how great documentation makes it easy for external contributors to join the fun. It’s a welcoming and inclusive community around it that makes your project alive. It’s all the best practices you can put in place to make it easier to participate, contribute and use your code. It’s clear governance and transparent processes. It's the credibility you build by being consistent, responsive, and honest, especially when things go wrong. Great code is only the beginning…
